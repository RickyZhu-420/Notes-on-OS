简答题简要

1. OS

操作系统是一种管理系统资源，控制程序执行，为用户提供方便接口的系统软件

操作系统包括处理器管理、存储器管理、设备管理、文件管理、作业管理模块

 

2. 进程

进程三状态概念和相互转换（见下文此处略）

创建进程的过程：申请空白 PCB、为新工序分配资源、初始化 PCB、将新进程插入就绪队列

原语是指由若干条指令组成的程序段，用来实现某个特定功能，在执行过程中不可被中断。OS中有些被进程调用的操作，如队列操作、对信号量的操作、检查启动外设操作等，一旦开始执行，就不能被中断，所以这些操作使用原语

进程控制块一般记录的信息：进程的标识信息、寄存器内容、进程状态、进程优先级、进程调度信息、进程所拥有的资源、进程打开的文件等信息

进程通信：

  • 高级通信：进程间要传送大量数据的时候，可以使用 OS 提供的高级通信工具，使用方便、高效

  • 高级通信归类：共享存储器系统、管道通信系统、消息传递系统以及客户机-服务器系统

    信箱机制也属于高级通信

  • 低级通信归类：信号、管道、FIFO

 

3. 调度

临界资源是在多进程或多线程环境下，由于资源数量有限，只能同时被一个进程或线程访问的资源

临界区是访问临界资源的程序片段

临界区管理的基本准则或条例：互斥原则、短小原则、避免持有锁原则、同步原则和避免死锁原则

系统的安全状态：所谓安全状态是指系统能按某种顺序如<P1,P2,...,Pn>（称<P1,P2,...,Pn>顺序为安全序列），来为每个进程分配其所需资源，直到最大需求，使每个进程都可顺序完成

在多道程序设计中，操作系统将 CPU 时间分割成很短的时间段（时间片），并在每个时间段中将 CPU 分配给不同的程序。这样，多个程序可以并发地执行，从而提高了系统的吞吐量和利用率

维持死锁的四个必要条件是：资源互斥、持有且等待、不可剥夺、循环等待

 

4. 存储器管理

存储管理模块基本功能：内存分配、地址重定位、内存保护与共享、虚拟内存等

缺页中断：程序访问一个尚未分配物理内存空间的地址时，系统会检测到这个情况，并触发异常

段式存储管理思想：把程序按内容或过程函数关系分成段，每段有自己的名字

快表：将作业的部分页表装入 cache 中即形成快表

联想存储器：存放快表的 cache 一般是联想存储器，联想存储器的最大特征是能在输入页号信息物理上立即输出对应的块号（若检索到的话）

 

5. I/O

独占设备的分配过程：分配设备、分配设备表、分配通道，每一步考虑所分配的东西忙不忙，三者成功才分配进程独占设备

SPOOLing 工作原理：若有进程要求对它打印输出时，SPOOLing 系统并不是将这台打印机直接分配给进程，而是在共享设备（磁盘或磁鼓）上的输出 SPOOLing 存储区内为其分配一块存储空间，进程的输出数据以文件形式存放于此。各进程的数据输出文件形成了一个输出队列，由输出 SPOOLing 系统控制这台打印机进程，依次将队列中的输出文件实际打印输出。在 SPOOLing 系统中，实际上并没有为任何进程分配，而只是在输入井和输出井中，为进程分配一存储区并建立一张 I/O 请求表。这样便把独占设备改造为共享设备

提示

SPOOLing 相关：

SPOOLING系统中包括了三个程序，分别是：输入程序、井管理程序、输出程序

 

6. 文件管理

文件目录表记录系统中所有文件的名字及其存放地址，也包括包括关于文件的说明信息和控制信息

基本内容有：文件名、文件内部标识、文件的类型、文件存储地址、文件的长度、访问权限、建立时间和访问时间等内容

文件系统的基本功能是：文件存储空间的管理、目录管理、文件读写管理和保护等

物理文件是文件系统在存储介质上的文件构造方式，常见物理文件：顺序文件、直接文件、索引文件、链接文件

 

7. 磁盘管理

磁盘的地址构成：柱面号，盘面号，扇区号

磁盘访问的时间组成：寻道时间 Ts、旋转延迟时间 Tr、传输时间 Tt

提示

文件相关：

逻辑文件是用户视角下的数据组织形式，根据业务需求和数据关系进行设计。物理文件则是计算机系统视角下的数据存储形式，根据存储介质和存储方式进行组织

常见物理文件：顺序文件、直接文件、索引文件、链接文件

常见逻辑文件：流式文件（无结构文件）、记录式文件（有结构文件）

1. OS 概述

计算机由硬件系统和软件系统组成，操作系统（OS）属于软件系统中的系统软件

OS 的功能有：处理器管理、存储管理、设备管理、文件管理、作业管理

OS 的特征有：并发性、共享性、虚拟性、异步性

提示

仔细区分并行、并发！

  • 并发是两个或多个事件可以在同一个时间间隔发生，用户看起来是每个程序都在运行，实际上是每个程序都交替执行

  • 并行是两个或多个事件可以在同一个时刻发生，多核 CPU 可以并行

  • 互斥共享：当资源被某程序占用后其他程序只能等待，例如打印机不能同时给两个程序用

  • 虚拟性：OS 能够把一个物理实体转变为若干个逻辑实体

    • 时分复用：资源在时间上进行复用，不同程序并发使用，多道程序分时使用计算机的硬件资源

    • 空分复用：例如硬盘充当虚拟内存，在逻辑上扩充了内存的容量

      提示

      硬盘即使一部分充当虚拟内存，那一部分也不能完全等同于内存。

      操作系统会将一些不常用的内存页（page）从物理内存移动到硬盘上的交换空间（swap space）或页面文件（page file）。这个过程称为“交换”（swapping）或“分页”（paging）

中断机制用于在多道批处理系统中让用户进行交互。

发生中断时：

  • CPU 转换为内核态

    提示

    CPU 有用户态和内核态两种状态

    处于内核态的 CPU 可以访问任意的数据，包括外围设备

  • 当前运行的进程暂停运行，由 OS 对中断进行处理

  • 处理完后返回原进程继续执行

中断分为内中断（亦称异常）和外中断

 

2. 进程

进程是主存中某个程序在某个数据集合上的一次执行的过程

进程控制块：用于描述和控制进程运行的通用数据结构,记录进程当前状态和控制进程运行的全部信息，是进程存在的唯一标识

进程与线程：

  • 线程：操作系统进行运行调度的最小单位

  • 进程：系统进行资源分配和调度的基本单位

区别与联系：

  1. 一个进程可以有一个或多个线程

  2. 线程包含在进程之中，是进程中实际运行工作的单位

  3. 进程的线程共享进程资源

  4. 一个进程可以并发多个线程，每个线程执行不同的任务

 

进程的三种基本状态是就绪态、执行态、阻塞态。此外还有进程的创建状态和终止状态。

提示

一个进程被唤醒进入就绪态

等待CPU资源的进程一定处于就绪态

原语：OS中一些进程执行过程中不允许被打断的程序段。一般用函数去描述

进程的各项控制操作必须由原语函数实现，因进程在各项操作中被打断可能会产生非预期结果

进程创建：检查系统是否满足新建进程的资源要求（除CPU状态外）。若满足则申请一空白PCB，填入新进程的相关信息（PID，代码地址，数据地址，进程状态（就绪态）等信息）插入相应的就绪队列

  • Windows创建进程：CreateProcess函数。Windows中进程无父子关系

  • Unix/Linux创建进程：fork函数。

    因Unix/Linux进程间有父子关系，使用时复制父进程PCB，之后修改子进程的ID，其余一致。子进程需要额外函数完成子进程执行代码的装入。任何进程有且只有一个父进程
